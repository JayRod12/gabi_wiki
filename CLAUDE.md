# Wiki Gabilondo — Schema

This file tells the LLM agent how this wiki is structured and what workflows to follow.

## Directory structure

```
raw/          # Source documents (immutable — never edit)
wiki/         # Markdown wiki pages (LLM writes these)
docs/         # Generated HTML (never edit by hand — always regenerated by build.py)
build.py      # Build script: wiki/*.md → docs/*.html
```

## Language

All wiki pages are written in **Spanish**. Source files in `raw/` may be in any language.

## Ingest workflow

When the user adds a new source to `raw/`:

1. Read the source file carefully.
2. Discuss key takeaways with the user if needed.
3. Update or create relevant wiki pages (typically 3–10 pages per source).
4. Always update `wiki/index.md` if new pages were added.
5. Append an entry to `wiki/log.md` using the format:
   ```
   ## [YYYY-MM-DD] ingest | <source filename>
   **Fuente:** `raw/<filename>`
   **Resumen:** <one paragraph>
   **Páginas creadas/actualizadas:** <comma-separated list>
   ```
6. Run the build: `pip install markdown && python build.py`
7. Commit all changes (wiki/ + docs/).

## Query workflow

When the user asks a question:

1. Read `wiki/index.md` to identify relevant pages.
2. Read those pages.
3. Synthesize and answer with citations to wiki pages and raw sources.
4. If the answer is valuable enough to keep (a comparison, analysis, or synthesis), create a new wiki page for it, run the build, and commit.

## Build

```bash
# First time only — creates a local venv
python3 -m venv .venv && .venv/bin/pip install markdown

# Every time you want to build
.venv/bin/python build.py
```

The build converts every `wiki/*.md` to `docs/*.html`, rewrites internal `.md` links to `.html`, and applies a consistent HTML template with sidebar navigation. Never edit `docs/` by hand.

## Publishing

Changes pushed to `main` that touch `wiki/**` or `build.py` trigger a GitHub Actions workflow (`.github/workflows/publish.yml`) that runs the build and deploys to GitHub Pages via the `gh-pages` branch. The published URL is:

```
https://jayrod12.github.io/gabi_wiki/
```

## Lint checklist

When asked to lint the wiki:
- Pages mentioned in `index.md` but not existing as files
- Pages that exist but are not listed in `index.md`
- Internal links pointing to non-existent pages
- Pages with no inbound links from other pages
- Contradictions between pages
- Sections marked "Por documentar" that could be filled from existing sources
- Missing cross-references between clearly related pages

## Conventions

- First line of each page: `# Page Title` (used as HTML title)
- Internal links: `[text](other_page.md)` — the build rewrites these to `.html`
- Sources always cited at the bottom of the page under `## Fuentes`
- Unknown information listed under `## Por documentar`
- Keep stubs — pages with little information — rather than leaving topics unlinked
